================================================================================
                       METADATA STRUCTURE (Per Function)
================================================================================

Each function entry contains 11 fields:

1. name                  - Function identifier (e.g., "np.array")
2. intent               - Clear description of purpose
3. category             - [main_category, subcategory]
4. relatedFunctions    - Array of similar function names
5. inputs              - Array of parameter objects with:
                        └─ name, type, meaning, constraints
6. outputs             - Object with type and meaning
7. preconditions       - Array of conditions before use
8. sideEffects         - Array of side effects
9. failureModes        - Array of error/exception modes
10. examples           - Array of usage examples with:
                        └─ description, code, output
11. category           - [main_category, subcategory] (duplicate for convenience)

Original Functions (30):
  ✓ FULL DETAILS: All 11 fields complete
  ✓ EXAMPLES: 3-5 detailed examples per function
